Problèmes Connus

  • Le support (version 1.4) du clavier est toujours plutôt minimal; il n'y pas moyen pour le moment d'envoyer des signaux et quelques combinaisons de touches ne sont pas supportées correctement (e.g., CTRL-ALT-DEL).
  • Curseur de la souris: à cause de problème JAVA, le curseur de la souris n'est pas correctement dessiné pendant par example le déplacement des fenêtres..
  • En raison des contraintes de sécurité plus rigides, Java Web Start de l'application assistée ne fonctionne pas avec les versions plus récentes de JAVA (>1.7.45) .

Propriétés du processus JAVA

Sous Windows, pour l'assistant j'utilise Launch4J pour envelopper la classe JAVA principale dans un fichier .exe. Vous pouvez toujours Configurez la JVM de chaque processus à l'aide du fichier .l4j.ini correspondant Le répertoire bin de votre Dayon! installation.

Vous pouvez par exemple configurer l'utilisation de la RAM JVM en tenant compte de la taille du cache et Si vous utilisez la méthode de compression LZMA (nécessite plus de RAM sur la côte de l'assistant).

L'option -server est disponible si vous utilisez un JDK (par opposition à un JRE).

Sous Unix, les mêmes options sont disponibles dans le script dayon.sh .

Répetoire d'accueil de Dayon!

Le répertoire .dayon est créé dans le répertoire référencé par le JAVA Propriété user.home et contient les préférences utilisateur enregistrées et le journal par défaut fichiers.

Somme de contrôle CRC

Sur le côté de l'assisté, l'écran est divisé en différentes zones appelées tuiles. Seules les tuiles qui ont été modifiées depuis la capture précédente sont envoyées sur le Côté assistant. Pour déterminer si une tuile est différente, je suis en train de calculer un code CRC (C'est-à-dire, une valeur entière unique représentant les pixels de la tuile) qui n'est pas parfaite Par souci de rapidité. Il se peut donc que certaines tuiles changées ne soient pas envoyées à l'assistant.

Jusqu'à présent, j'ai détecté ce problème lors de tests forts pour très peu de pixels. Visuellement, Je n'ai rien remarqué de sérieux. Mais au cas où les choses vont fou, vous pouvez ensuite redémarrer L'aide ou avant d'essayer l'action reset ( l'icône orange de la foudre ) Qui devrait effacer toutes les données en cache et renvoyer une capture plein écran à partir de zéro.

Compteurs de statistiques

La barre d'état du cadre assistant affiche un ensemble de compteurs.

  1. Bande passante réseau
  2. Compression Ratio: combien de fois la capture initiale (diff only) a été comprimé
  3. Number of Tiles: le nombre de carreaux étant également transmis sur le réseau ou servis depuis le cache.
  4. Number of Skipped Capture: le nombre de captures d'écran qui ont été ignorées En raison d'un taux trop élevé (c'est-à-dire, une valeur de cotation faible) pour la CPU. Pour minimiser cette vous devez ralentir le taux de capture en utilisant une valeur plus grande.
  5. Number of Merged Capture: le nombre de captures d'écran fusionnées avant d'être transmis. Ceci est dû à un taux de capture trop élevé pour le courant méthode de compression. Pour minimiser ce nombre, vous devez ralentir la capture taux et / ou changer la méthode de compression en utilisant un plus rapide.
