Bekannte Einschränkungen

  • Auch mit Version 1.4 ist die Tastaturunterstützung noch immer ziemlich minimalistisch; Es gibt zZt. keine Möglichkeit gewisse Tastenkombinationen wie etwa (e.g., CTRL-ALT-DEL) an den entfernten Computer zu senden.
  • Mauszeiger: technologisch (JAVA) bedingt, wird der Mauszeiger während dem Vergrössern oder Bewegen von Fenstern mitunter nicht sauber dargestellt.
  • Wegen verschärften Sicherheitseinstellungen, funktioniert der Java Web Start der Applikation des Unterstützten mit neueren JAVA Versionen (>1.7.45) leider nicht mehr.

JAVA Prozess Eigenschaften

Unter Windows, wird für den Assistenten Launch4J verwendet, um die JAVA-Hauptklasse in eine .exe-Datei zu packen. Sie können die JVM für diesen Prozess konfigurieren, indem Sie die entsprechende .l4j.ini Datei im bin Verzeichnis des Dayon! Installationsverzeichnisses anpassen.

Sie können dort etwa den der JVM zugewiesenen Arbeitsspeicher anpassen. Je nach Grösse des Puffers und der verwendeten Kompressionsmethode. (LZMA ist auch in dieser Hinsicht etwas resourcenhungriger).

Die -server Option ist bloss verfügbar, wenn Sie das JDK verwenden. (an Stelle der JRE).

Unter Linux sind diese Optionen im dayon.sh Skript konfigurierbar.

Dayon! Heimverzeichnis

Das Verzeichnis .dayon wird im via JAVA-Property definierten Benutzerverzeichnis user.home erstellt und enthält nebst den gespeicherten Einstellungen auch die Logdateien.

CRC Checksumme

Auf der Seite des Unterstützten wird der Bildschirm in Kacheln genannte Bereiche aufgeteilt. Bloss veränderte Kacheln werden über das Netzwerk zum Assistenten gesendet. Um zu entscheiden ob sich eine Kachel seit der letzten Erfassung verändert hat, wird ein CRC-Code berechnet (ein eindeutiger Wert für die in einer Kachel enthaltenen Pixel). Dieses Verfahren ist hinsichtlich der Verarbeitungsgeschwindigkeit nicht die optimale Lösung. Es kann mitunter vorkommen, dass einige veränderte Kacheln nicht an den Assistenten gesandt werden.

Bis jetzt konnte rein durch intensives Testen Verfälschungen von einigen Pixeln entdeckt werden. Visuell konnte bis anhin nichts ernsthaft störendes festgestellt werden. Sollte in der Praxis in dieser Hinsicht Mal etwas aus dem Ruder laufen, kann entweder die Applikation des Unterstützten neu gestartet, oder dessen Puffer zurück gesetzt werden. (oranges Pfeilsymbol).

Statistiken in Statusleiste

Die Statusleiste des Assistenten zeigt folgende Zähler an:

  1. Netzwerkbandbreite
  2. Kompressionsverhältnis: wie oft der Inhalt der initial erfassten Kacheln (bloss Differenz) komprimiert wurde
  3. Anzahl Kacheln: Anzahl übermittelter Kacheln (inkl. Puffer-Treffer).
  4. Anzahl übersprungener Bildschirmaufnahmen: Anzahl Bildschirmaufnahmen, welche wegen eines zu kurzen Aktualisierungsintervall (für die entsprechende Hardware) übersprungen worden sind. Um diese Anzahl zu reduzieren, erhöhen Sie das Erfassungsintervall.
  5. Anzahl kombinierter Bildschirmaufnahmen: Anzahl Bildschirmaufnahmen, welche vor der Übertragung kombiniert worden sind. Dies ist ein Anzeichen für ein zu kurzes Erfassungsintervall für die gewählte Komprimierungsmethode. Am einfachsten das Erfassungsintervall erhöhen und/oder eine schnellere Kompressionsmethode wählen.
